
# Question Evaluation Rubric

This rubric evaluates AI-generated questions across four dimensions: technical depth, cognitive independence, technology usage, and question intent.

---

## 1. Technical Depth (Conceptual Complexity)

Measures the level of technical knowledge and abstraction required to meaningfully answer the question.

**0 – Non-Technical**

* No programming or technical concepts
* Examples: clubs, career advice, definitions without implementation

**1 – Foundational**

* Basic concepts or definitions
* Syntax-level or introductory explanations
* Example: frontend vs backend

**2 – Applied**

* Practical use of known tools, algorithms, or patterns
* Simple projects or standard problem-solving
* Example: implementing WebSockets, basic dynamic programming

**3 – Advanced**

* Optimization, system design, scalability, or tradeoffs
* Requires analysis beyond standard usage
* Example: runtime optimization, architecture decisions

**4 – Expert**

* Research-level or highly specialized knowledge
* Novel constraints, system internals, or theoretical depth
* Example: distributed systems internals, compiler optimizations

---

## 2. Cognitive Independence

Measures how much reasoning, decision-making, and problem decomposition is expected from the question asker, rather than being fully outsourced to the AI.

**1 – Outsourced Thinking**

* Requests full solutions or code generation
* No constraints or reasoning provided
* Example: “Write this program for me”

**2 – Guided Execution**

* Asks AI to implement, fix, or debug with limited analysis
* Example: “Fix this bug”, “Implement X using Y”

**3 – Analytical**

* Requests comparison, explanation of tradeoffs, or optimization reasoning
* Example: “Which approach is better and why?”

**4 – Metacognitive**

* Reflects on strategy, learning process, or problem-solving approach
* Example: “How should I approach solving these problems?”

---

## 3. Technology Usage Level

Evaluates whether modern technologies are referenced and how meaningfully they are incorporated.

**0 – None**

* No technologies or tools mentioned

**1 – Name-Dropping**

* Mentions tools or platforms without context or purpose
* Example: “Should I use AWS?”

**2 – Contextual Use**

* Technologies are tied to a specific function or problem
* Example: WebSockets for real-time updates, Firebase for auth

**3 – Architectural Awareness**

* Demonstrates understanding of limitations, scaling, or alternatives
* Example: AWS IAM design, database tradeoffs

---

## 4. Question Intent (Primary Purpose)

Classifies the main goal of the question. Only one primary intent should be selected.

* **Foundational Learning** – Understanding basic concepts
* **Problem-Solving / Implementation** – Building or implementing a feature
* **Optimization / Scaling** – Improving performance, reliability, or scale
* **Learning Strategy** – Improving how one learns or prepares
* **Career / Social** – Networking, clubs, job preparation

---

### Usage Notes

* Each question is scored independently across all dimensions.
* Higher technical depth does not imply higher quality; intent matters.
* This rubric enables comparison of questions generated by different AI personas (e.g., MIT vs MADA).
